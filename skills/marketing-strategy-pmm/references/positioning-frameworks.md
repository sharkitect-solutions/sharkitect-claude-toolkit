# Positioning Frameworks Reference

## April Dunford 5-Step Positioning Canvas

### Step 1: List True Competitive Alternatives

What would your best-fit customers do if your product didn't exist?

**Categories to include (most teams miss 2-3 of these):**
- Direct competitors (same category, same buyer)
- Adjacent solutions (different approach, overlapping use case)
- Platform incumbents (broader tool that includes your capability)
- Status quo solutions (spreadsheets, email, manual processes)
- DIY / build in-house
- Do nothing (live with the problem)

**Process:**
1. Interview 10-15 recent closed-won customers
2. Ask: "What were you doing before us? What else did you evaluate?"
3. Cluster responses -- you'll find 3-5 real alternatives
4. Rank by frequency: the most-mentioned alternative is your primary competitive context

**Validation rule:** If >50% of customers say "spreadsheets" or "nothing," your primary competitor is inertia, not a named product. Position accordingly.

### Step 2: Isolate Unique Attributes

What capabilities do you have that NO alternative provides?

**The 6-Month Test:** For each candidate attribute, ask: "Could our top competitor replicate this in 6 months?" If yes, it's a feature, not a defensible attribute. Dig deeper.

**Attribute categories:**
- Technical architecture (e.g., real-time vs batch processing)
- Data advantage (e.g., proprietary dataset, network effects)
- Workflow integration (e.g., embedded in existing tools vs standalone)
- Domain expertise (e.g., built by practitioners, not generalists)
- Economic model (e.g., usage-based vs seat-based fundamentally changes buyer math)

**Output format:**
```
Attribute: [Name]
What it is: [Technical description]
Why alternatives can't replicate: [Structural reason]
Time to replicate: [Estimate -- must be >12 months to count]
```

### Step 3: Map Attributes to Customer Value

Each attribute must connect to a value that customers experience AND can articulate.

**Value mapping chain:**
```
Attribute (what you have)
  --> Capability (what it enables)
    --> Benefit (what the customer experiences)
      --> Measurable outcome (what changes in their business)
```

**Example:**
```
Attribute: Real-time collaboration engine
  --> Capability: Multiple users edit simultaneously
    --> Benefit: No version conflicts, no waiting
      --> Outcome: 40% faster project completion (measured across 50 customers)
```

**Rules:**
- Every chain must end in a measurable outcome. "Better experience" is not an outcome.
- Outcomes must be validated with customer data. If you can't measure it, it's a hypothesis.
- Each attribute should map to 1-2 value chains max. If it maps to 5, you haven't focused enough.

### Step 4: Define Best-Fit Customers

Who cares MOST about the value from Step 3?

**Best-fit is NOT your total addressable market.** It's the narrowest segment where you win fastest and retain longest.

**Identification method:**
1. Pull closed-won deals from last 12 months
2. Sort by: (a) shortest sales cycle, (b) highest NPS, (c) lowest churn, (d) highest expansion
3. Find the intersection -- customers who rank high on 3+ dimensions
4. Identify shared characteristics: industry, size, tech stack, buying trigger, org structure

**Validation criteria:**
- Segment must contain >100 addressable accounts (too small = no market)
- Segment must be reachable through identifiable channels
- Segment must have a common buying trigger you can target in messaging

### Step 5: Choose Your Market Category

The category frames how buyers evaluate you. Choosing wrong here undermines everything else.

**Category strategy decision matrix:**

| Signal | Strategy | Execution |
|---|---|---|
| You win >40% of competitive deals in an existing category | Head-to-head | Compete on your unique attribute, not on feature count |
| You win on ONE dimension but lose on breadth | Subcategory (big fish, small pond) | Carve out "[Category] for [Segment]" -- e.g., "CRM for agencies" |
| Existing categories don't describe what you do AND you have $50M+ budget | Create new category | Requires 3-5 years of market education investment |
| None of the above | Reframe, don't create | Position as existing category + modifier: "AI-powered [Category]" |

**Category creation warning signs (don't do it if):**
- You're pre-Series B
- You can't commit 3+ years to category education
- Analysts already have a category that's "close enough"
- Your sales team can't explain the category in one sentence

### Trend Layer (Step 5b)

After choosing category, layer on 1-2 trends that create urgency:
- Trend must be real and acknowledged by your buyers (not just VCs)
- Trend must make your approach MORE relevant, not just more timely
- Avoid trend-chasing: "AI-powered" only works if AI is architecturally central, not a feature

## Positioning Statement Templates

### Internal Positioning Document (for team alignment)

```
For [best-fit customer description]
Who [key buying trigger / pain point]
[Product name] is a [market category]
That [key differentiated value]
Unlike [primary competitive alternative]
We [unique attribute that delivers the value]
We know this because [proof point: metric, customer quote, or data]
```

### External One-Liner (for website, pitch, collateral)

```
[Product] helps [target customer] [achieve outcome] by [unique approach].
```

**Rules for the one-liner:**
- Must be understandable by someone who's never heard of your product
- Must contain a specificity that separates you from competitors
- Must NOT contain jargon, buzzwords, or superlatives ("best," "leading," "revolutionary")
- Test: Read it to 5 people outside your company. If 3+ can't explain what you do, rewrite.

### Board/Investor Positioning (for fundraising context)

```
We're building [category] for [segment].
Today's solutions [specific limitation of alternatives].
We [unique approach] which means [differentiated outcome].
We've proven this with [traction: customers, revenue, growth rate].
The market is [TAM with source] and [trend] is accelerating demand.
```

## Value Proposition Hierarchy

Map your positioning into a hierarchical value structure that cascades through all marketing and sales materials:

```
LEVEL 1 -- Value proposition (one sentence)
  What you do + for whom + key differentiation

LEVEL 2 -- Benefit pillars (3-4 max)
  Each pillar = one major area of value
  Each must be independently compelling
  Each must be defensible vs competitors

LEVEL 3 -- Feature evidence (2-3 per pillar)
  Specific capabilities that deliver each pillar's promise
  Each linked to a measurable outcome

LEVEL 4 -- Proof points (per feature)
  Customer quotes, metrics, case studies, third-party validation
  Must be specific and verifiable
```

**Cascade rule:** Everything at Level 3 must roll up to a Level 2 pillar. Orphan features (cool but don't support a pillar) should be deprioritized in messaging, even if engineering is proud of them.

**Consistency rule:** If a sales rep's talk track (Level 4) contradicts the value proposition (Level 1), fix it immediately. Messaging inconsistency erodes buyer trust faster than any competitor action.
