# Client Report Templates

## Weekly Status Report Template

### Structure

```
PROJECT STATUS REPORT
[Project Name] | Week of [Date] | Overall Status: [GREEN/YELLOW/RED]

EXECUTIVE SUMMARY (2-3 sentences max)
[One sentence on overall progress. One sentence on the most important development
this week. One sentence on what's coming next.]

THIS WEEK'S PROGRESS
- [Deliverable/milestone]: [Status] - [One line on what was accomplished]
- [Deliverable/milestone]: [Status] - [One line on what was accomplished]
- [Deliverable/milestone]: [Status] - [One line on what was accomplished]

KEY METRICS
| Metric              | Target | Actual | Trend |
|---------------------|--------|--------|-------|
| [Primary KPI]       |        |        | up/down/flat |
| [Secondary KPI]     |        |        | up/down/flat |
| [Budget consumed]   |        |        | on/over/under |

BLOCKERS & RISKS (if any)
- [Blocker]: [Impact if unresolved] | [Owner] | [Target resolution date]

NEXT WEEK'S PRIORITIES
1. [Priority with expected outcome]
2. [Priority with expected outcome]
3. [Priority with expected outcome]

DECISIONS NEEDED (if any)
- [Decision]: [Options] | [Needed by date]
```

### Writing Guidance

- **Executive summary:** Write this last. It should stand alone -- a reader who stops here understands project health.
- **Progress section:** Use action verbs (completed, launched, resolved), not activity verbs (worked on, continued, ongoing). Clients pay for outcomes, not effort.
- **Metrics table:** Maximum 5 metrics. Include trend direction. Prior period comparison is more informative than raw numbers.
- **Blockers:** Only list items that require client awareness or action. Internal blockers go in internal tracking, not client reports.
- **Tone:** Confident, factual, forward-looking. Avoid hedging language ("we think," "hopefully," "should be"). State what is known and what is planned.

## Monthly Progress Report Template

### Structure

```
MONTHLY PROGRESS REPORT
[Client Name] | [Month Year] | Overall Status: [GREEN/YELLOW/RED]

EXECUTIVE SUMMARY
[3-5 sentences covering: overall progress against plan, key achievements,
significant challenges addressed, outlook for next month.]

PROGRESS AGAINST OBJECTIVES
| Objective                   | Target    | Actual    | Status | Notes |
|-----------------------------|-----------|-----------|--------|-------|
| [Strategic objective 1]     |           |           |        |       |
| [Strategic objective 2]     |           |           |        |       |

MONTH HIGHLIGHTS
[Narrative paragraph: 3-4 key accomplishments with quantified results.
Connect each accomplishment to a client business objective.]

CHALLENGES & RESOLUTIONS
| Challenge              | Impact            | Resolution/Plan           | Status   |
|------------------------|-------------------|---------------------------|----------|
| [Issue encountered]    | [Business impact] | [What was/will be done]   | Resolved/In progress |

BUDGET STATUS
| Category         | Budgeted | Spent to Date | Remaining | Projected Total |
|-----------------|----------|---------------|-----------|-----------------|
| [Labor/hours]   |          |               |           |                 |
| [Tools/tech]    |          |               |           |                 |
| [Other]         |          |               |           |                 |
| **Total**       |          |               |           |                 |

NEXT MONTH OUTLOOK
- [Key initiative with expected outcome]
- [Key initiative with expected outcome]
- [Risks or decisions on the horizon]

APPENDIX
[Detailed data tables, analytics screenshots, supporting documentation]
```

### Writing Guidance

- **Highlights narrative:** Tell a story, not a list. Connect activities to outcomes to business impact. "We redesigned the checkout flow (activity), reducing cart abandonment by 18% (outcome), which projects to $42K additional quarterly revenue (business impact)."
- **Challenges section:** Never present a challenge without a resolution or plan. Unresolved items get an owner and target date. This section proves you manage problems proactively.
- **Budget status:** Always show projected total, not just spent-to-date. Clients need to know if you will be over or under at completion. Surprises at project end are relationship-ending events.

## Retainer Activity Report Template

### Structure

```
RETAINER ACTIVITY REPORT
[Client Name] | [Month Year] | Utilization: [X]%

VALUE DELIVERED THIS MONTH
[2-3 sentences: top outcomes delivered, not hours spent. Lead with impact.]

ACTIVITY SUMMARY
| Activity                    | Hours | Deliverable/Outcome               |
|-----------------------------|-------|-----------------------------------|
| [Strategic initiative]      |       | [What was produced/achieved]      |
| [Ongoing maintenance]       |       | [What was maintained/improved]    |
| [Ad hoc requests]          |       | [What was resolved]               |
| **Total**                  | X/Y   | Y = allocated hours               |

HOURS BREAKDOWN
| Category         | Hours | % of Total | Notes                    |
|-----------------|-------|------------|--------------------------|
| Strategic work   |       |            | Proactive, goal-aligned  |
| Maintenance      |       |            | Keep-the-lights-on       |
| Ad hoc requests  |       |            | Unplanned client requests|
| Internal (admin) |       |            | Reporting, planning      |

ROLLOVER/BANK STATUS (if applicable)
| Period        | Allocated | Used | Banked | Cumulative Bank |
|--------------|-----------|------|--------|-----------------|

NEXT MONTH PRIORITIES
1. [Strategic initiative with expected outcome]
2. [Ongoing item with performance target]
3. [Proposed initiative if hours are available]

RECOMMENDATIONS
[One paragraph: proactive suggestions for next month based on trends observed.
This section distinguishes strategic partners from task executors.]
```

### Writing Guidance

- **Value delivered:** Always the first section. Never start with hours consumed. If the first thing the client reads is "87 of 120 hours used," they evaluate you as a cost center. If the first thing they read is "3 campaigns launched, 2,400 leads generated," they evaluate you as a growth partner.
- **Hours breakdown by category:** This prevents "where did the hours go?" conversations. When a client sees 30% on ad hoc requests, it opens a conversation about process improvement -- a conversation that benefits both parties.
- **Recommendations:** This single section justifies a premium retainer rate. Vendors report what happened. Partners recommend what should happen next.
